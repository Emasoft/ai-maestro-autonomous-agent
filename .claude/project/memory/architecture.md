`ai-maestro-autonomous-agent` is the **mandatory role-plugin for AUTONOMOUS-titled
agents** in the AI Maestro ecosystem. It is a governance/behavior plugin (no
runtime service): it ships ONE main-agent persona plus a small set of skills that
enforce workspace isolation, forbid cross-agent state mutation, encode the R6
communication graph (AMP), and wire the agent into the PRRD/TRDD kanban and the
global janitor-hosted markdown memory system. The agent belongs to no team,
serves the user directly, and coordinates with MANAGER (and peer AUTONOMOUS
agents) over AMP.

## Parts map
- **Persona** — `agents/ai-maestro-autonomous-agent-main-agent.md`: the full
  AUTONOMOUS behavior contract (writable scope, forbidden actions, R6 comm graph,
  approval tiers, solo-mode dialog loops, memory protocol, startup checklist).
- **Skills** — `skills/`: `ai-maestro-autonomous-governance` (rules checklist),
  `ai-maestro-autonomous-workspace-isolation` (writable-scope edge cases),
  `ai-maestro-autonomous-prrd-trdd-kanban` (the PRRD/TRDD lifecycle). Memory
  operations are the GLOBAL `/janitor-memory-{recall,write,update}` skills — this
  plugin no longer ships per-plugin memory skills (TRDD removed them; see below).
- **Publish pipeline** — `scripts/publish.py`: the canonical version-bump +
  validate (ruff/mypy/pytest + CPV `--strict` remote) + commit + push + release
  pipeline. The real pre-push gate is `.githooks/pre-push` (process-ancestry,
  auto-generated by publish.py).
- **Tests** — `tests/`: structural tests over the persona + skills frontmatter
  (`test_skills_structural.py` enumerates the shipped skills).
- **Design** — `design/`: `requirements/PRRD.md` (project rules) +
  `tasks/`/`proposals/`/`archived/`/`refused/` (TRDD lifecycle).
