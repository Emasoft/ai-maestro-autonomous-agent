# TRDD-5c21e4a0 — Address CPV RC-PIPELINE-DRIFT-001 (pipeline-canon drift)

## Context

CPV `--strict` flags `RC-PIPELINE-DRIFT-001` (a **non-blocking WARNING**, 5 of the
8 advisories at v1.4.0) on `publish.py`, the two workflows, `cliff.toml`, and
`.markdownlint.json` — suggesting migration to the latest CPV canon via
`/cpv-upgrade-plugin` (or `uvx cpv-remote-validate standardize <plugin> --fix
--force-templates`). Surfaced during the v1.4.0 memory-migration validation;
pre-existing (also present at v1.3.3, which deliberately did not migrate).

This is a **proposal** (Tier-2: touches CI workflows + the release pipeline) —
NOT yet authorized. Awaiting USER/MANAGER decision.

## The split: a SAFE subset vs a RISKY overwrite

**SAFE (recommend doing) — workflow hardening, no pipeline-logic change:**
- **SHA-pin the 2 third-party actions** — `astral-sh/setup-uv@v5` and
  `peter-evans/repository-dispatch@v4` are currently tag-pinned; `~/.claude/rules/gh-actions.md`
  requires third-party actions (outside `actions/`/`github/`) pinned to a full
  commit SHA (use `pinact`). First-party `actions/*@vN` pins are fine as-is.
- Optionally add the canon's **actionlint + commitlint gates** and a **macOS
  matrix** (this plugin already declares `runtime-targets: [macos, linux]`).
- These are additive/low-risk and align with our own security rule.

**RISKY (recommend NOT doing via `--force-templates`):**
- `--force-templates` **overwrites `publish.py`** with the generic canon template.
  This plugin's `publish.py` carries hard-won custom logic — the M11 version-sync
  across plugin.json + pyproject + persona + README + uv.lock (took THREE releases
  to get right; see archived TRDD-e7281b7e), the `language_bump_version` path, and
  the `[^\S\n]*$` README-badge regex. A blind force-overwrite would likely regress
  these. Any canon migration of `publish.py` must be done as a careful 3-way merge
  (keep our custom bump/sync logic), validated by a real `--dry-run` + a test
  release, NOT a template clobber.

## Plan (if approved)

1. SHA-pin the 2 third-party actions with `pinact` (or manual SHA lookup); keep
   tag comments. Validate workflows still parse (actionlint).
2. (Optional) add actionlint + commitlint gates + macOS matrix to CI.
3. Re-run CPV `--strict`; confirm RC-PIPELINE-DRIFT-001 count drops.
4. Publish a patch (v1.4.1). **Do NOT** run `--force-templates` against `publish.py`
   without a reviewed 3-way merge.
